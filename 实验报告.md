### 1.2 实验原理

#### 1.2.1 TensorFlow Lite 简介

**TensorFlow Lite** 是 TensorFlow 针对移动设备、嵌入式设备和 IoT 设备的轻量级解决方案。相比完整的 TensorFlow，它具有以下优势：

| 特性 | 说明 |
|------|------|
| **轻量级** | 模型体积小，占用空间少 |
| **低延迟** | 针对移动端 CPU/GPU 优化，推理速度快 |
| **低功耗** | 计算量小，省电 |
| **跨平台** | 支持 Android、iOS、Linux、MCU 等 |

**工作原理：**
```
TensorFlow 模型 (SavedModel)
        ↓ TFLiteConverter 转换
TensorFlow Lite 模型 (.tflite)
        ↓ FlatBuffer 格式
移动端推理（TFLite Interpreter）
```

#### 1.2.2 CameraX 架构

**CameraX** 是 Android Jetpack 的一部分，采用**用例（Use Case）**驱动的设计：

| 用例 | 功能 |
|------|------|
| **Preview** | 在屏幕上显示相机实时画面 |
| **ImageAnalysis** | 提供每一帧图像用于分析（ML 推理） |
| **ImageCapture** | 拍摄高质量照片（本项目未使用） |

**生命周期绑定：** CameraX 会自动绑定到 Activity/Fragment 的生命周期，自动管理相机的打开和关闭，避免资源泄漏。

#### 1.2.3 MVVM 架构

项目采用 **Model-View-ViewModel** 架构模式：

```
┌─────────────────────────────────────────────────┐
│                   View (UI)                       │
│  MainActivity.kt + XML Layout (RecyclerView)     │
└──────────────────┬──────────────────────────────┘
                   │ 观察 LiveData
         ┌─────────▼──────────┐
         │   ViewModel         │
         │ RecognitionListVM   │
         │ (持有 LiveData)     │
         └─────────┬──────────┘
                   │ 封装数据
         ┌─────────▼──────────┐
         │    Model            │
         │ FlowerModel(TFLite) │
         │ Recognition 数据类  │
         └────────────────────┘
```

### 1.5 核心代码分析

#### 1.5.1 图像格式转换流程

从相机获取的原始图像是 **YUV_420_888** 格式（Y 亮度 + U/V 色差），需要转换为 RGB：

```
ImageProxy (YUV_420_888)
    ↓ YuvToRgbConverter.yuvToRgb()
    ↓ 使用 RenderScript 在 GPU 上并行处理
Bitmap (ARGB_8888)
    ↓ TensorImage.fromBitmap()
TensorImage
    ↓ FlowerModel.process()
TensorFlow Lite 推理
```

关键代码位于 `YuvToRgbConverter.kt`：
- 使用 Android RenderScript 在 GPU 上并行处理图像转换
- 性能远高于 CPU 单线程转换

#### 1.5.3 ViewModel 与 LiveData

`RecognitionListViewModel` 管理识别结果数据：

```kotlin
class RecognitionListViewModel : ViewModel() {
    // 内部可变 LiveData（仅 ViewModel 可修改）
    private val _recognitionList = MutableLiveData<List<Recognition>>()

    // 对外暴露不可变 LiveData（UI 仅可读）
    val recognitionList: LiveData<List<Recognition>> = _recognitionList

    fun updateData(recognitions: List<Recognition>) {
        // 后台线程安全地更新 LiveData
        _recognitionList.postValue(recognitions)
    }
}
```

在 `MainActivity` 中观察：

```kotlin
recogViewModel.recognitionList.observe(this, Observer {
    viewAdapter.submitList(it)  // 自动更新 RecyclerView
})
```

**为什么这样设计？**
- ViewModel 独立于 Activity 生命周期，屏幕旋转时数据不丢失
- LiveData 是生命周期感知的，只在 Activity 处于活跃状态时推送更新
- 符合 Android 官方推荐的架构模式

### 2.2 实验原理

#### 2.2.1 卷积神经网络（CNN）

图像分类使用**卷积神经网络（CNN）**，核心层包括：

| 层类型 | 作用 |
|--------|------|
| **卷积层（Conv2D）** | 提取图像特征（边缘、纹理、形状、语义） |
| **池化层（Pooling）** | 下采样，减少计算量，增强特征鲁棒性 |
| **全连接层（Dense）** | 将提取的特征映射到类别输出 |
| **Softmax** | 将输出转为概率分布（总和为 1） |

#### 2.2.2 迁移学习

**为什么用迁移学习？**

| 问题 | 解决方案 |
|------|---------|
| 训练数据不足（只有几千张图） | 使用在大型数据集（ImageNet，1400万张图）上预训练的模型 |
| 训练时间太长（需要数天） | 冻结预训练权重，只训练顶层分类器 |
| 需要大量算力（GPU/TPU） | 基于已学习的通用视觉特征进行微调 |

**MobileNetV2 架构**：
- Google 专为移动端设计的轻量级 CNN
- 深度可分离卷积（Depthwise Separable Convolution）减少参数量
- 相比传统 CNN，参数减少 80-90%，精度损失很小
- 模型大小仅 3-5MB，适合移动端部署

```
输入 (224, 224, 3)
    ↓
MobileNetV2 预训练模型（1000 类分类）
    ↓ 丢弃顶层分类层
GlobalAveragePooling2D
    ↓ 全局平均池化（替代全连接层）
Dropout(0.2)        ← 防止过拟合
    ↓
Dense(5, softmax)   ← 5 类花卉分类
```

#### 2.2.3 TensorFlow Lite 转换流程

```
Keras 模型 (.h5)
    ↓ TFLiteConverter.from_keras_model()
TFLite 转换器
    ↓ optimizations = [tf.lite.Optimize.DEFAULT]
    ↓ 权重量化（float32 → int8）
    ↓ 模型大小减少 75%，速度提升
TensorFlow Lite 模型 (.tflite)
    ↓ 放入 Android 项目 ml/ 目录
ML Model Binding → FlowerModel Kotlin 类
```

#### 步骤 4：构建迁移学习模型

使用 MobileNetV2 作为基础模型进行迁移学习：

```python
num_classes = len(class_names)  # 5

# 1. 数据增强层（训练时随机变换图像，防止过拟合）
data_augmentation = keras.Sequential([
    layers.experimental.preprocessing.RandomFlip("horizontal"),
    layers.experimental.preprocessing.RandomRotation(0.1),
    layers.experimental.preprocessing.RandomZoom(0.1),
])

# 2. 加载预训练的 MobileNetV2（不包含顶层分类器）
base_model = tf.keras.applications.MobileNetV2(
    input_shape=(img_height, img_width, 3),
    include_top=False,    # 不包含顶层分类器
    weights='imagenet'    # 使用 ImageNet 预训练权重
)

# 3. 冻结基础模型（不更新预训练权重）
base_model.trainable = False

# 4. 构建完整模型
inputs = keras.Input(shape=(img_height, img_width, 3))
x = data_augmentation(inputs)                                    # 数据增强
x = tf.keras.applications.mobilenet_v2.preprocess_input(x)      # 归一化
x = base_model(x, training=False)                               # 特征提取
x = layers.GlobalAveragePooling2D()(x)                          # 全局平均池化
x = layers.Dropout(0.2)(x)                                      # Dropout 防止过拟合
outputs = layers.Dense(num_classes, activation='softmax')(x)     # 5 类分类

model = keras.Model(inputs, outputs)

# 显示模型结构
model.summary()
```

#### 步骤 9：转换为 TensorFlow Lite 格式

这是**连接桌面训练与移动端部署**的关键步骤：

```python
# 1. 创建转换器（从 Keras 模型）
converter = tf.lite.TFLiteConverter.from_keras_model(model)

# 2. 应用优化（量化压缩）
# 这会将 32 位浮点数量化为 8 位整数
# 模型大小减少 75%，推理速度提升，精度损失可接受
converter.optimizations = [tf.lite.Optimize.DEFAULT]

# 3. 执行转换
tflite_model = converter.convert()

# 4. 保存为 .tflite 文件
tflite_model_path = 'FlowerModel.tflite'
with open(tflite_model_path, 'wb') as f:
    f.write(tflite_model)

print(f"TFLite 模型已保存: {tflite_model_path}")
print(f"模型大小: {os.path.getsize(tflite_model_path) / 1024 / 1024:.2f} MB")
```

### 2.6 实验心得与总结

1. **迁移学习的威力**：在小数据集上（仅 3000 多张图），通过迁移学习能达到 >90% 的准确率。如果从零训练，准确率可能只有 50-70%。

2. **数据增强的重要性**：随机翻转、旋转、缩放等数据增强操作显著降低了过拟合风险。不使用增强时，验证准确率可能低 10-15%。

3. **MobileNetV2 是移动端的绝佳选择**：模型仅 3-5MB，但精度接近大型 CNN。相比 ResNet50（~100MB），非常适合移动部署。

4. **TensorFlow Lite 量化优化效果显著**：优化后模型大小减少 75%，推理速度提升 2-3 倍，精度损失 <1%。

5. **GPU 加速的价值**：在支持 GPU 的 Android 设备上，GPU 委托（Delegate）可带来 2-5 倍推理加速。代码中的自动检测机制确保了兼容性。

6. **端到端工作流**：从数据收集 → 模型训练 → 模型转换 → 移动端部署，完整的 AI 应用开发流程已经打通。
